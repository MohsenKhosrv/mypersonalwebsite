Long-form case study content lives here, one file per case study.

- Filename must match the case study's `id` from `lib/case-studies.ts`, e.g. `demand-forecasting.md`.
- Plain paragraphs, separated by a blank line.
- Optional subheadings: put `## Heading text` on its own line/paragraph.
- No other markdown syntax is parsed (no bold/italic/links/lists) — keep it to headings and paragraphs.

Until a case study's file exists here, its Projects card and full-story page fall back to the
short `problem` / `whyItMattered` / `solution` / `keyImpact` fields from `lib/case-studies.ts`.
